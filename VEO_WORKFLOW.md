# Veo + Google Flow 기반 비디오 자동 제작 워크플로우

이 문서는 정적 이미지 줌인 연출(Ken Burns 효과)을 배제하고, **Veo(Image-to-Video)** 및 **Google Flow** 자동화를 사용하여 역동적인 4~5초 분량의 비디오 클립들을 연결하여 최종 영상을 제작하는 새로운 표준 워크플로우를 정의합니다.

---

## 🔄 전체 제작 프로세스

```
[1] 대본 세분화 (5~6초 단위)
         ↓
[2] 이미지 생성 프롬프트 작성 (워터마크 차단 포함)
         ↓
[3] Google Flow / Image Generator로 베이스 이미지 생성 (1024x1024)
         ↓
[4] flow_automator.py 실행 (비디오 클립 자동 생성 및 다운로드)
         ↓
[5] make_video.py 실행 (TTS 오디오 매칭, 자막 합성 및 최종 렌더링)
```

---

## 📑 단계별 상세 지침

### 1단계: 대본 세분화 (Segmentation)
*   전체 대본을 낭독 속도 기준 **5~6초 단위**의 짧은 씬(Scene)으로 분할합니다.
*   각 씬마다 화면전환이 발생하여 영상이 정적으로 늘어지는 현상을 방지합니다.

### 2단계: 이미지 프롬프트 작성 및 생성
*   각 씬의 핵심 메타포를 시각화하는 프롬프트를 영어로 작성합니다.
*   **원천 워터마크 차단 규칙:** AI 이미지 생성 모델이 하단에 임의의 텍스트나 바이너리 코드를 그리지 않도록 모든 프롬프트 끝에 아래의 부정 키워드를 삽입합니다:
    ```
    No watermark, no text, no binary code, no letters, no signatures.
    ```
*   생성된 원본 이미지는 가로/세로 비율을 유지하여 **1024x1024** 크기의 `assets/images/scene_X.png`로 저장합니다.

### 3단계: Veo를 활용한 비디오 클립 생성 (Google Flow)
*   생성한 `scene_X.png`를 기반으로 **Veo(Image-to-Video) 모형**을 이용하여 **3~4초 분량의 역동적인 비디오 클립**(`assets/videos/scene_X.mp4`)을 생성합니다.
*   이 과정은 Playwright 자동화 스크립트인 `python flow_automator.py`를 통해 Google Chrome의 로그인 프로필 컨텍스트를 사용하여 Google Flow 웹 페이지 상에서 완전 자동으로 실행 및 다운로드됩니다.
*   **단순 이미지 무빙 금지:** 단순히 베이스 이미지 자체를 평면적으로 흔들거나 확대하는 수준에 그쳐서는 안 되며, 3~4초 동안 시간의 흐름에 따라 **장면의 변화와 카메라 워크가 어우러진 복합적인 동영상 작업**이 유도되도록 프롬프트를 정교하게 설계해야 합니다.
*   **비디오(모션) 프롬프트 필수 구성 요소:**
    *   **카메라 다차원 무빙 (Camera Work):** 1차원적인 줌인/패닝을 넘어 플라이스루(Flythrough), 궤도 비행(Orbital shot), 극적인 돌리 줌(Dolly zoom), 180도 회전(Rotate to reveal) 등 공간감을 극대화하는 카메라 워크 지시.
    *   **장면의 변화 및 다른 장면 삽입 (Scene Transition / Insertion):** "카메라가 패닝하여 옆에 있는 새로운 사물/인물을 비춘다", "스마트폰 화면 속 캐릭터가 화면 밖으로 튀어나온다", "배경이 낮에서 밤으로 매끄럽게 흐른다" 등 3~4초 내에 시각적 변화나 보조 장면 삽입 묘사.
    *   **동작 및 상호작용 (Interaction):** 캐릭터가 사물을 만지거나, 시선을 돌리거나, 뇌 속 분자들이 활발히 춤을 추는 등 실제 픽셀 단위의 완전한 물리 동작 변화 지시.
*   *주의:* 이제 단순 줌인 효과 코드로 때우는 방식은 완전히 금지하며, 스크립트 작성 시 위의 다이내믹한 씬 모션 프롬프트를 상세한 영어로 같이 짜서 제공해야 합니다.

### 4단계: 비디오 컴파일 및 자막 인코딩 (`make_video.py`)
*   **속도 동기화:** `assets/audio/scene_X.mp3`(TTS)의 재생 길이에 맞춰, `assets/videos/scene_X.mp4` 비디오 클립의 배속을 자동 조절(`MultiplySpeed`)하여 동기화합니다.
*   **한글 자막 렌더링:** Windows 환경에서 폰트 깨짐(tofu 현상)을 방지하기 위해 반드시 절대 경로인 `C:\Windows\Fonts\malgun.ttf` 파일 경로를 사용합니다.
*   자막 영역이 잘리지 않도록 텍스트 박스 높이를 200px 이상 확보하고 마진을 맞춥니다.
