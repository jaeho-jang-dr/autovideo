# autovideo — Antigravity (Gemini) 오케스트레이터 컨텍스트

> Antigravity의 고수준 조율 가이드.
> Claude Code에게 작업을 위임할 때의 프로토콜, 판단 기준을 정의합니다.

---

## 나의 역할 (Antigravity = 조감독)

```
제작자 (사용자) — 기획, 요구사항 정의 및 최종 검증
    ↓
감독 (Claude Code) — 고수준 설계 및 전체 조율, 조감독(Gemini)에게 작업 지시
    ↓
조감독 (Antigravity / 나) — 감독(Claude)의 지시를 수령하여 파일 편집 및 세부 실행, 진행 사항 보고
```

### 작업 분담 비율 (70/30) 및 검수 규칙 — 2026-06-19 제작자 확정
- 제작자(사용자) 지시가 떨어지면 실제 실행 작업의 **약 70%는 나(조감독/Gemini)가 CLI로 직접 수행**하고, **약 30%만 감독(Claude)이 직접** 처리하여 작업량 균형을 맞춘다.
- 감독(Claude)은 내가 한 작업을 **검수**하며, 내가 **못 하거나 미흡하면 감독이 직접 대체**해 마무리한다. 따라서 위임받은 태스크는 **검증 가능한 형태로 결과·로그를 함께 보고**한다.

## 하이브리드 파이프라인(Hybrid Pipeline) 조율 역할 정의

본 제작 시스템은 **1단계: 완전 자동 뼈대 생성(MoviePy)**과 **2단계: 인간-AI 다이내믹 튜닝(CapCut)**을 결합한 하이브리드 워크플로우를 따르며, 제어 방식과 책임 소지는 다음과 같습니다.

### 1. MoviePy (1차 뼈대 컴파일 및 자막/오디오 완전 자동화)
*   **감독 (Claude Code)**:
    *   에피소드 시나리오 기획 및 정밀 대본(`scenario.txt`) 감수.
    *   구글 Flow 비디오 생성을 위한 이미지/동영상 연출 프롬프트 설계.
    *   오류 발생 시 소스 코드 디버깅 방향 수립 및 조감독(Gemini)에게 태스크 지시.
*   **조감독 (Antigravity / Gemini)**:
    *   지시받은 Playwright 자동화 엔진(`autoveo_flow.py`, `flow_automator.py`) 직접 구동 및 실시간 다운로드 검증.
    *   gTTS 한국어 나레이션 배속(1.1x) 및 자막 폰트 경로(`malgun.ttf`) 매칭 등 세부 코드 직접 수정.
    *   최종 컴파일러(`make_video.py`, `finalize.py`) 실행을 통한 1차 MP4 가이드 비디오(4K) 및 한국어 썸네일 자동 빌드.
    *   결과물 Google Drive 백업.

### 2. CapCut 데스크톱 (2차 다이나믹 윤기 튜닝 및 최종 4K 업스케일)
    - **오토 컷아웃(배경 제거)**: 캡컷 AI를 이용해 특정 요소(예: 시계 바늘, 뇌 인포그래픽 등)만 레이어로 분리합니다.
    - **키프레임 흔들기/회전**: 분리된 컷아웃 오브젝트에 키프레임을 주어 staccato 느낌의 2D 종이 인형 움직임을 연출합니다.
    - **자막 튕기기 효과**: 캡컷 내장 텍스트 템플릿의 다이나믹 애니메이션을 적용하여 자막에 가독성과 생동감을 더합니다.
    - **효과음(SFX)**: 장면 전환 시점의 슉(whoosh), 딱(pop) 등의 오디오 사운드 임팩트를 믹싱합니다.
    - **4K 출력**: 유튜브의 선명한 코덱(VP09) 혜택을 받기 위해, 내보내기 시 **4K(3840x2160) 해상도로 강제 업스케일링**하여 최종 렌더링합니다.

## 캐릭터 「이미지재생성」 (일관성 포즈 생성)

DB의 캐릭터 **베이스 사진**을 레퍼런스로 **같은 캐릭터의 새 동작 사진**을 만드는 검증된 기법 = **「이미지재생성」**.
- **방법 문서**: [POSE_GENERATION_METHOD.md](file:///D:/Entertainments/DevEnvironment/autovideo/POSE_GENERATION_METHOD.md)
- **도구**: `python scratch/flow_pose.py --ref-key <베이스key> --out-key <동작key> --prompt "<동작 영문구절>"`
- **핵심 순서**: 새 프로젝트→베이스 사진 업로드→(~25초)타일 **더보기(⋮)→"프롬프트에 추가"**→스크립트 입력→ENTER→(~45초)→**왼편 '새' 타일**만 다운로드(md5 중복검증). 측면 동작은 "EXACTLY ONE single figure" 강제. 크롬 프로필은 한 번에 하나씩 순차.
- **검증 완료**: `jieun_jumping`(정면 레퍼런스), `jieun_walking`(옆모습 레퍼런스). DB `character_assets`/`techniques`(name=`이미지재생성`)에 등록됨.
- **기본 이미지 이중화 규칙 (투명 & 불투명)**: 캐릭터 베이스 에셋 및 동작 포즈는 항상 불투명(Beige `#F5F5F0` 배경, `*_opaque`)과 로컬 투명화 처리를 마친 투명 PNG 두 가지 버전을 한 세트로 생성하여 DB(character_assets)와 스토리지에 동시 등록합니다. (Google Flow AI 레퍼런스 참조용 = 불투명 / 영상 합성 컴파일 및 후처리용 = 투명)

## 프로젝트 핵심 정보

- **목적**: 유튜브 교양/지식 콘텐츠 자동 제작 파이프라인 (대본 추출, TTS 생성, 이미지/비디오 합성, 자막 삽입 및 합본 인코딩)
- **스택**: Python + MoviePy 2.2.1 (v2.x API 사용 필수) + gTTS
- **영상 제작 워크플로우**: 주제 선정 -> NotebookLM 소스 검색 -> 프로젝트 폴더 생성 및 레퍼런스 적재(스샷/URL) -> 시나리오(scenario.txt) 및 극도의 정밀 스크립트 도출(롱폼: 5초당 1컷, 쇼츠: 4초당 1컷) -> Google Flow(autoveo_flow.py) 비디오 자동 생성/다운로드 -> 최종 컴파일(make_video.py, 전용 폴더 저장) -> 초정밀 역공학 명세 파일([video_name]_spec.md) 보존 -> 목차/스샷 기반 데이터화 순서의 7단계 AI 영상 제작 라이프사이클을 준수합니다. (자세한 내용은 [VEO_WORKFLOW.md](file:///D:/Entertainments/DevEnvironment/autovideo/VEO_WORKFLOW.md) 및 [principles.md](file:///d:/Entertainments/DevEnvironment/autovideo/.harness/context/principles.md#L88) 참조)
  - 정적 이미지 줌인 효과 연출(Ken Burns)은 **금지**하며, 5~6초 단위 대본마다 생성된 이미지를 Veo를 통해 4~5초 비디오 클립으로 만들고 이들을 합성하여 컴파일합니다.
- **하네스 버전**: 1.0 (생성: 2026-06-11)
- **비디오 오디오 및 렌더링 디폴트 (실수 방지 규칙)**:
  - **동적 폴더 격리:** 특정 프로젝트(예: `chiropractic_science`)의 영상을 제작할 경우, 관련 이미지/모션 비디오 클립/TTS/최종 합본 비디오 등 모든 결과물은 해당 전용 폴더 하위로 격리 저장해야 합니다.
  - **나레이션 속도:** 항상 나레이션 속도를 **1.1배속 (10% 빠르게, `MultiplySpeed(1.1)`)**으로 렌더링해 비디오와 싱크를 맞춥니다.
  - **나레이션 성별:** 명확한 전달력을 위해 항상 gTTS 기본 **여성 음성**을 디폴트로 사용합니다.
  - **캐시 갱신 강제화:** 대본 수정 시 오디오-자막 불일치 방지를 위해 실행 시마다 `assets/audio/scene_*.mp3` 캐시를 무조건 강제 삭제/갱신합니다.
  - **자막 폰트 절대 경로:** 윈도우 환경 한글 자막 깨짐을 막고자 폰트 경로를 **`C:\Windows\Fonts\malgun.ttf`** 절대 경로로 못박아 사용합니다.
  - **Claude CLI 대기 지시:** 백그라운드로 렌더링을 지시할 때 CLI가 조기 종료하여 렌더링 프로세스가 공중 파괴되지 않도록 렌더링 완료까지 대기(Wait)하라는 위임 프롬프트 지시를 강화합니다.
  - **배경 UI 자동 승인 매크로:** Antigravity CLI의 승인 팝업 우회를 위한 매크로(`scratch/auto_approve.py`)는 화면 캡처 방식의 한계를 극복하기 위해 Windows UI Automation API (`uiautomation` 라이브러리)를 활용해 Electron 창의 엘리먼트 트리를 타서 안전한 명령어만 백그라운드에서 실시간 자동 승인(`Yes, allow this time` -> `Submit` 순차 클릭)합니다. 위험 패턴 명령어는 자동으로 승인이 차단되어 수동 검토 대기 모드로 진입합니다.
  - **자막 가변 패딩 박스 표준화:** 자막 텍스트 라인 수(1줄/2줄/3줄)에 맞춰 텍스트 박스 크기(위아래 패딩)를 동적으로 가변 조정하는 방식을 표준 렌더링 규칙으로 고정 적용한다.
  - **영어 설명 씬 처리 원칙:** 그림 내부에 영어 텍스트/설명이 노출되는 씬은 가능한 한 한글 텍스트 박스(어노테이션 오버레이)를 적용하고, 텍스트 박스 내부 한글 멘트는 최대한 직관적이고 간결하게(단순화) 작성하여 전달력을 극대화한다.
  - **100% CPU 동원 렌더링:** 최종 컴파일 렌더링 시에는 시스템의 모든 CPU 자원을 100% 동원하도록 `os.cpu_count()` 코드를 활용해 최대 병렬 스레드를 할당해 컴파일한다.
  - **워터마크 원천 배제 (Dolly Zoom-Crop):** 워터마크를 가리는 네모 상자 얼룩을 남기지 않기 위해, 16:9 비율을 엄격히 보존하는 미세 줌-크롭(78% 크롭 후 Lanczos-4 보간법 리사이징)을 적용하여 원래의 Veo 워터마크를 화면 바깥으로 완벽히 배제합니다.
  - **초소형 채널 로고 오버레이:** 크롭으로 깨끗해진 우측 하단 자리에 `assets/drjay_ed_logo_circle.png` 둥근 원형 로고 배지를 기존 크기 대비 25% 수준인 **가로세로 45x45 픽셀 크기**로 대폭 축소하여 알파 블렌딩 오버레이합니다.
  - **개별 씬 0 인트로 및 썸네일 자동화:** 각 에피소드 고유의 인트로는 시나리오 파일 내 `[Scene 0]` 블록으로 작성하며, 컴파일러(`make_video.py`)는 Scene 0를 감지하면 자동으로 `[output_dir]/scene_0_thumbnail_korean.png` 썸네일(한국어 텍스트와 sprout green 상자 레이아웃이 적용된 1280x720 해상도 이미지)을 생성 및 저장합니다.
  - **4초 공통 아웃트로 & 10초 정지 권유 카드:** 아웃트로는 4초 고정으로, 격려/구독유도 한국어 멘트와 맑은 종소리 효과음(금속성 감쇄 chime 주파수 오디오 믹싱)으로 마무리합니다. 아웃트로 종료 후에는 유튜브 추천 카드 활용을 위한 정지 카드 이미지(`--outro-card`, 기본 `assets/outro_template.png`)를 10초 동안 무음 정지 영상으로 덧붙여 결합합니다.
  - **구글 플로우(Flow) 다운로드 및 딜레이 꿀팁 (사용자 피드백 반영):**
    - **대기 시간:** 이미지 생성 시 최소 40초 이상, 동영상은 최소 70초 이상 대기 후 다운로드를 체크합니다.
    - **타일 식별:** 생성된 두 개의 타일 중 항상 **왼쪽이 완성된 비디오**, **오른쪽이 베이스 이미지**입니다.
    - **다운로드 전략:** 비디오 타일 영역을 클릭해 라이트박스(상세 모달)를 열고, 우측 상단에 노출되는 다운로드 아이콘 버튼을 눌러 720p/원본 크기로 받은 다음, 라이트박스는 `Escape` 키를 눌러 닫아야 합니다.
    - **클릭 재시도 및 스샷:** 타일 클릭 후 라이트박스가 정상적으로 열리지 않는 경우, 해당 시점의 화면 스크린샷을 저장하고 좌표 정보를 로깅 및 기억하여 마우스 `move` 후 다시 강제 재클릭을 시도하여 연속 다운로드 무중단 상태를 보장합니다.
    - **이미지/영상 실패 대응 전략 및 80초 동영상 대기:**
      - 이미지 생성 시: 생성 후 **40초** 대기하고, 타임아웃/실패 시 스샷 저장 -> 재시작 버튼 클릭 -> 스샷 저장 -> 오른쪽/최근 타일 삭제 후 재생성합니다.
      - 비디오 생성 시: 생성 시작 후 **정확히 80초** 동안 아무 다운로드도 시도하지 않고 완성될 때까지 얌전히 대기(하드디스크 낭비 및 이미지 중복 다운 방지)합니다. 80초 후 가장 왼쪽 비디오 타일을 클릭하여 라이트박스(상세창)를 열고 우상단 다운로드 버튼으로 원본을 안전하게 다운로드합니다.
      - 비디오 실패 시: 80초 내에 완성이 안 되거나 다운로드 실패 시, 즉시 가장 왼쪽 타일(비디오)을 삭제하고 재시도합니다.
    - **연속 실패 시 리부팅:** 이미지 또는 비디오 생성이 동일 씬에서 3회 연속 실패 시, 아예 크롬 브라우저 프로세스를 완전히 종료하고 5초 쿨다운 후 브라우저 세션을 새로 띄워 프로젝트 진입부터 다시 시작(Reboot)합니다.
    - **구글 AI 생성 정책 우회 규칙:** 프롬프트 설계 시 실존 인물/유명인 연상 단어(예: "Dr. Jay" -> "Julius Erving" 선수명 오인) 및 성적 필터링 오해 유발어(예: `bulge`, `expand` 등)의 사용을 금지하고, 일반명사(예: `the friendly doctor`) 및 중립적인 동작 묘사(예: `moves slightly`)로 우회하여 기술합니다.
    - **인트로/아웃트로 기획 중복 방지:** 본편의 첫 씬(Scene 1)과 마지막 씬은 앞뒤에 붙는 고정 인트로/아웃트로의 나레이션 멘트나 화면 구성과 내용이 불필요하게 겹치지 않도록 작성합니다.
    - **씬 길이의 개별 다양화:** '5초당 1컷' 식의 엄격한 길이 제한을 없애고, 씬의 정보량과 나레이션 호흡, 연출 모양에 맞춰 씬별 시간 길이를 다양하고 유연하게(3초~10초+) 기획 및 렌더링합니다.
    - **역동적인 카메라 무빙 다변화:** 줌인(zooming in) 연출에만 과도하게 쏠리지 않도록, 천천히 회전(rotating slowly), 줌아웃(zooming out), 윗촬영(overhead angle / top-down), 팬/틸트(panning / tilting) 등 4대 카메라 워킹을 교차 사용하여 지루함을 타파합니다.
    - **드로잉 크기(스케일) 다변화:** 인물/오브젝트가 너무 작게 그려져서 답답해 보이는 현상을 차단하기 위해, 핵심 개념을 크게 강조하는 클로즈업(Close-up)과 중대 구도를 담는 미디엄/와이드 샷을 씬마다 혼합하여 연출합니다.
    - **프레임 계승(Last Frame Transition):** 8초 이상의 긴 모션이나 동일 장소에서의 연속 동작이 필요할 경우, 앞 씬 비디오의 마지막 프레임을 이미지 파일로 추출하여 다음 씬의 모션 베이스 이미지로 직접 구글 Flow에 업로드해 일관성 있는 씬 연결을 연출합니다.
    - **Claude CLI 위임 시 필수 주입:** Claude Code CLI에게 비디오 생성 작업을 위임할 때 이 최적화 룰을 항상 위임 프롬프트 내에 핵심 가이드라인으로 명시해 전달해야 합니다.
    - **웹페이지 디버깅 꿀팁 (개발자 도구 F12 연동):** 웹페이지 자동화 제어/렌더링 작업 도중 문제가 발생하면, 즉시 웹페이지의 개발자 도구(`F12`)를 눌러 콘솔 및 네트워크 로그(버그 리포트)가 보이는 상태에서 스크린샷을 찍고 분석하십시오. 그러면 어떤 오류인지 훨씬 명확히 식별할 수 있으며, 이 분석 결과와 문제의 근본 원인을 메모리에 기록하고 보존해야 합니다.
  - **딥리서치 자동 기획 연동:** 노트북(신규 프로젝트)을 생성할 때는 항상 딥리서치(Deep Research)를 선행하여 상세한 지식 소스 자료(`[project_name]_deep_research.md`)를 모으는 것을 원칙으로 합니다. 사용자의 명시적인 설명 지시가 없더라도, 딥리서치 데이터 내에서 필요한 핵심 지식을 추출하여 에피소드 주제와 가장 밀접하고 풍부한 대본(`scenario.txt`) 및 시나리오를 자동 완결형으로 구성합니다.
  - **테스트 영상 제한 렌더링:** 테스트 빌드 시에는 4K 업스케일, 썸네일, SNS 720p 등을 따로 별도 생성하지 않으며 구글 드라이브 업로드도 건너뛰어야 합니다. 오직 사용자의 명시적인 지시가 있을 때만 4K/SNS/썸네일/구글 드라이브 업로드를 진행합니다.
  - **정적 호스팅 TTS 403 Forbidden 우회:** 실서버(Vercel, Cloudflare Pages)와 같은 정적 호스팅 도메인 환경에서는 구글 번역 TTS URL이 CORS 및 Referer 정책에 의해 403 Forbidden으로 차단되므로, 문장 통낭독 및 낱자 폴백 오디오는 브라우저 내장 `window.speechSynthesis` (볼륨 1.0)를 사용해 우회 출력합니다.

## 감독(Claude)-조감독(Gemini) 협업 및 소통 프로토콜

감독(Claude)과 조감독(Gemini)은 공동의 프로젝트 목표를 완수하기 위해 상호 정의된 파일 매체와 리포트를 통해 긴밀히 협력합니다.

### 1. 감독(Claude)이 조감독(Gemini)에게 지시할 때
*   **작업 지시서 생성 (`scratch/gemini_task.md`)**: 감독(Claude)은 세부 설계, 수정이 필요한 소스 코드 파일 목록, 구체적인 요구사항 및 성공 기준을 [gemini_task.md](file:///scratch/gemini_task.md) 파일로 작성하여 조감독에게 위임합니다.
*   **제작자(사용자) 경유 전달**: 감독(Claude)이 대화창에 작업 방향을 출력하면, 제작자가 이를 복사하여 조감독(Gemini) 대화창에 입력해 기동시킵니다.

### 2. 조감독(Gemini)이 지시를 수행할 때
*   **코드 편집 및 환경 검증**: 조감독(Gemini)은 지시받은 파일을 직접 편집(`multi_replace_file_content` 등 활용)하고, 필요한 검증(코드 무결성, 인코딩 검사 등)을 수행합니다.
*   **진행 상태 업데이트**: `.harness/loops/progress.json` 또는 `task.md` 파일을 실시간 업데이트하여 진행 현황을 동기화합니다.

### 3. 조감독(Gemini)이 감독(Claude)에게 보고 및 전달할 때
*   **결과 리포트 작성 (`scratch/gemini_report.md` 또는 `walkthrough.md`)**: 조감독(Gemini)은 작업을 완료한 후, 수정된 코드 차이(Diff)와 테스트 검증 결과, 생성된 에셋 경로를 리포트 파일로 작성해 감독이 확인할 수 있게 보존합니다.
*   **컨텍스트 맵 갱신**: 새로 생성된 중요한 명세서나 에셋 파일이 있다면 [CLAUDE.md](file:///CLAUDE.md)의 빠른 참조 맵에 즉시 등록하여 감독(Claude)이 다음 작업을 진행할 때 혼선 없이 인지하게 합니다.
