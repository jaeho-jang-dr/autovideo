## P9 — 비디오 오디오 및 렌더링 디폴트 규칙 (실수 방지 체크리스트)

*   **나레이션 속도 디폴트:** 항상 원래 나레이션 음성 속도 대비 **10% 빠르게 (1.1배속, `MultiplySpeed(1.1)` 적용)** 렌더링을 진행한다. 이에 맞춰 비디오 속도도 1.1배속으로 자동 가속 매칭된다.
*   **나레이션 성별 디폴트:** 항상 명확하고 신뢰도 높은 **여성 음성**(`gTTS` 한국어 기본 목소리)을 사용한다.
*   **TTS 캐시 불일치 버그 방지:** 대본 변경 시 구버전 오디오 캐시가 사용되어 자막과 엇갈리는 현상을 막기 위해, 렌더링 실행 전 반드시 기존 오디오 캐시 파일(`assets/audio/scene_*.mp3`)을 **강제로 삭제**하거나, 스크립트가 실행 시 무조건 오디오 파일을 덮어쓰도록(Overwrite) 한다.
*   **자막 폰트 절대 경로화:** Windows 환경에서 한국어 폰트가 깨져서 블록 표시(`☒☒☒☒`)로 나오는 에러를 방지하기 위해, 자막 폰트 경로는 반드시 윈도우 시스템 절대 폰트 경로인 **`C:\Windows\Fonts\malgun.ttf`**를 하드코딩하여 사용한다.
*   **Claude CLI 동기식 렌더링 완료 대기:** 백그라운드로 렌더링을 지시할 때 CLI 세션이 조기 퇴근하여 파이썬 프로세스가 폭사하는 문제를 방지하기 위해, 명령에 반드시 "파이썬 실행 완료 및 동영상 생성이 성공할 때까지 대기(Wait)"하도록 지시하거나 직접 파이썬 프로세스를 동기적으로 실행하고 감시한다.
*   **uiautomation 기반 자동 승인 매크로:** 팝업 승인 우회 매크로(`scratch/auto_approve.py`)는 `uiautomation` 라이브러리를 사용해 백그라운드에서 창을 제어합니다. 단, 파괴적인 위험 명령(rm, del, git push --force 등)은 자동으로 승인을 차단하고 수동 입력을 대기시킵니다.

## P10 — AI 영상 제작 라이프사이클 (표준 워크플로우)

**프로젝트별 주제 조사부터 시나리오 기획, 자동화 영상 빌드, 특이점 명세화 및 목차 데이터화까지 아래 7단계의 라이프사이클을 철저하게 준수한다.**

1. **주제 및 소스 조사**: 사용자가 대주제를 설정하면, NotebookLM에 관련 최신 지식 및 백그라운드 소스를 입력하여 활용할 수 있도록 준비합니다.
2. **폴더 구성 및 레퍼런스 적재**: 동영상 제목을 정하고 해당 전용 프로젝트 폴더(예: `chiropractic_science/`)를 루트에 생성합니다. 레퍼런스 사이트의 스크린샷, 참고 동영상의 스크린샷, 리버스 엔지니어링 대상 동영상 URL 등 입력 자료들을 해당 폴더에 적재합니다.
3. **시나리오 및 스크립트 도출**: 
   * **극도의 정밀 스크립팅 Mandate**: 시나리오 대본(`scenario.txt`)과 씬별 모션/이미지 프롬프트 스크립트를 작성할 때, 대충 타협하거나 생략하지 않고 씬 하나하나마다 카메라 무빙, 인물 동작, 컷아웃 질감, 구도를 극도로 세부적이고 정밀하게 묘사하여 작성해야 합니다.
   * **표준 컷 배분 규칙**:
     * **유튜브 롱폼 (예: 7분 영상)**: 낭독 속도에 맞춰 **평균 5초당 1컷 (7분 기준 84컷 내외)**의 이미지를 기본 규격으로 삼아 대본 문장을 배분합니다.
     * **유튜브 쇼츠 (숏폼)**: 트렌디하고 빠른 시청 이탈 방지를 위해 **평균 4초당 1컷 (60초 기준 15컷 내외)**을 표준 규격으로 삼습니다.
4. **자동화 영상 생성**: 사용자가 영상 길이(Duration) 및 클립(이미지) 개수를 지정하면, `autoveo_flow.py` 등을 가동하여 Google Flow 브라우저 자동화로 이미지/모션 비디오를 자동 생성한 뒤, 미리 생성해 둔 전용 프로젝트 폴더로 직접 다운로드받습니다.
5. **통합 컴파일**: 다운로드된 개별 모션 비디오 클립들을 TTS(gTTS 1.1배속) 및 자막(맑은 고딕 절대 경로)과 병합하여 하나의 최종 영상으로 컴파일하고, 동일한 전용 프로젝트 디렉토리에 저장합니다.
6. **특이점/명세서 문서 보존**: 
   * **초정밀 역공학 아카이빙 Mandate**: 다음 프로젝트나 유지보수 시 참고할 수 있도록, 이번 동영상 제작에 적용한 구체적인 연출/스크립트 특이점 및 리버스 엔지니어링 특이점을 **동영상 파일명과 동일한 명세 파일(예: `[video_name]_spec.md`)로 작성하여 해당 전용 폴더에 보존**합니다. 연출 분석은 오디오 템포, 비주얼 텍스처, 모션 다이내믹스, 폴리 사운드, 타이포그래피 등의 다차원 영역에서 아주 세밀하게 쪼개서 분석하고 저장해야 합니다.
7. **데이터화 및 시각화**: 보존한 특이점 및 스크립트를 목차(TOC)와 스크린샷 등을 가동하여 보기 편하게 정리하고, 프로젝트 데이터 아카이브로써 축적합니다.
