# Human-in-the-Loop 게이트 정의

> AI가 반드시 사람에게 확인을 받아야 하는 시점.

---

## Gate #1 — 비가역적 작업

**트리거**: 되돌릴 수 없거나 리소스를 소비하는 작업 전

```
예: 외부 API 비용 발생 작업, 데이터 삭제, 이메일 발송
```

**AI가 해야 할 일**:
1. 작업 내용과 이유 설명
2. 영향 범위 명시
3. 사용자 "예" 확인 후 진행

---

## Gate #2 — 프로덕션 배포

**트리거**: 배포 실행 전

```
AI가 할 일: 테스트 결과 공유 + 변경 파일 목록 + "배포할까요?" → 확인
자동 허용: 사용자가 명시적으로 "배포해줘" 라고 했을 때
```

---

## Gate #3 — 콘텐츠 대규모 삭제/변경

**트리거**: 3개 이상 파일 동시 대규모 수정, 중요 콘텐츠 삭제 전

```
AI가 할 일: 목록 제시 → 이유 설명 → 영향 범위 → 승인 요청
```

---

## Gate #4 — 환경변수 / 인증 변경

**트리거**: .env 파일 수정 전

```
AI가 할 일: 변경 내용 설명 → 사용자 직접 수정 요청 (AI가 키 노출 금지)
```

---

## Gate #5 — 연속 에러

**트리거**: 2회 이상 연속 실패, 예상치 못한 데이터 손실

```
AI가 즉시 할 일:
1. 작업 중단
2. 에러 내용 전체 보고
3. 복구 옵션 제시 (git revert 등)
4. 사용자 지시 대기
```
