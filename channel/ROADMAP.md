## 0. 비전 한 장 요약

TED-Ed가 **YouTube 채널 + ed.ted.com(레슨 사이트)** 두 축으로 굴러가듯,
drjay-ed도 **① YouTube 채널 + ② DrjayEd.com(이중언어 레슨 허브)** 을 함께 키운다.
영상 자산은 한 번 만들어 **한국어·영어** 양쪽으로, **YouTube와 자체 사이트** 양쪽에 노출한다.

```
            ┌──────────────────────────────────────────┐
            │        content.db  (SQLite, 단일 원천)      │
            │   카테고리5 · 에피소드24 · 씬 · 상태 · KR/EN  │
            └───────────────┬───────────────┬───────────┘
                            │ export/sync   │ 제작 파이프라인
                 ┌──────────▼─────────┐  ┌──▼─────────────────────┐
                 │  DrjayEd.com        │  │  영상 제작 (Veo/MoviePy) │
                 │  Cloudflare Pages   │  │  → render_kr / render_en │
                 │  + D1 + R2          │  └──┬─────────────────────┘
                 └──────────┬──────────┘     │ 업로드
                            │ 임베드           ▼
                            └──────────►  YouTube 채널 (KR/EN)
```

## 2. 웹사이트 아키텍처 (Cloudflare, 확장성 우선)

**추천 스택 — 우리 SQLite를 그대로 잇는 가장 자연스러운 길:**
- **도메인:** Cloudflare Registrar에서 `drjayed.com` 등록(원가 ~$10/년, 네임서버 자동).
- **프런트:** **Astro**(콘텐츠 사이트 최적·이중언어 i18n 내장) → Cloudflare **Pages** 배포. (대안: Next.js)
- **DB:** Cloudflare **D1**(서버리스 SQLite). `content.db` 스키마를 **그대로 이식** → 카탈로그가 곧 사이트.
- **에셋:** Cloudflare **R2**(썸네일·다운로드). 영상 본편은 YouTube 임베드(스토리지 절약).
- **API/자동화:** 필요 시 Cloudflare **Workers**.

**사이트 구조(ed.ted.com 벤치마크):**
- 홈 → 카테고리(CA/MD/IMP/LM/GEN) → 레슨 페이지(YouTube 임베드 + 한/영 대본 + 추후 퀴즈/노트).
- URL i18n: `drjayed.com/ko/...` · `drjayed.com/en/...`.
- **확장성:** content.db에 에피소드를 추가/상태변경 → export → D1 동기화 → 사이트 자동 반영.

**핵심 원칙:** *항상 켜진 공개 사이트는 Cloudflare 엣지가 책임진다. 내 컴퓨터(맥미니)를 서버로 둘 필요 없음.*

## 3. 상시 제작 스테이션 (M5 Mac Mini Pro)

- **역할 분리:** 사이트=Cloudflare(엣지). **맥미니=제작/업로드 스테이션** — Veo/Flow 자동화, MoviePy 렌더링, TTS, YouTube 자동 업로드를 24/7 무인 수행.
- **⚠️ 플랫폼 주의:** 현재 파이프라인은 **Windows 전용**(폰트 `C:\Windows\Fonts\malgun.ttf`, PowerShell, Session 0 ImageGrab). macOS 이전 시 **포팅 필요** — 폰트 경로(AppleSDGothicNeo/번들 폰트), 화면 자동화(Flow는 Playwright로 크로스플랫폼 가능), 스크린그랩 대체.
- **권장 순서:** 지금 Windows에서 워크플로를 완성·검증 → 안정화 후 맥미니로 포팅. (지금 맥 구매를 기다릴 필요 없음.)

## 4. YouTube 채널 만드는 법 & 주의점

### A. 개설 절차
1. **Google 계정** 준비(채널 전용 권장) → **2단계 인증** 필수.
2. YouTube → 계정 → **채널 만들기** → **브랜드 계정(Brand Account)** 로 생성(공동관리·양도 가능, 개인계정과 분리).
3. **채널명** `drjay-ed` + **핸들** `@drjayed` 설정. 도메인·핸들·채널명 **3개 통일**.
4. **채널아트:** 배너 2560×1440(안전영역 1546×423), 프로필 800×800(원형 로고 보유), 워터마크.
5. **About/설명:** 슬로건·이중언어 소개·DrjayEd.com 링크.

### B. 업로드 기본 규격
- **해상도** 1080p+ / **종횡비** 16:9 / **썸네일** 1280×720(2MB 이하).
- **자막:** 영상별 KR/EN 자막 트랙 업로드(우리는 번인 자막 + 별도 .srt 둘 다 가능).
- **이중언어 운영 — 2가지 길:**
  - **(A) 분리 영상 2개:** `render_kr`·`render_en`를 각각 업로드. 알고리즘·시장 분리, 단순. **초기 권장.**
  - **(B) 다국어 오디오 트랙:** YouTube **multi-language audio** 기능으로 1영상에 한/영 음성+자막. 조회수 합산 유리하나 운영 복잡.

### C. 꼭 알아야 할 주의점
- **수익화(YPP) 요건:** 구독자 **1,000명** + 최근 12개월 공개 시청 **4,000시간** (또는 Shorts 90일 1,000만 뷰). 천천히 누적되는 게임.
- **AI 콘텐츠 공시 의무:** 사실적 합성/변형 콘텐츠는 업로드 시 **'altered/synthetic content' 표시** 필요. 우리는 **본인 육성 내레이션 + 본인의 의학적 통찰**을 핵심에 둬 '진정성 있는 오리지널'로 포지셔닝 → 정책·신뢰 양면에서 유리.
- **저작권:** BGM·이미지·폰트 **상업적 사용 가능 라이선스만**. (Veo 생성물·자체 TTS·CC0/구매 음원 사용, YouTube 오디오 보관함 활용.)
- **반복·대량생성 정책:** 저품질 양산 콘텐츠는 수익화 거절 대상 → **편당 오리지널 가치(의학 렌즈·직접 해설)** 를 반드시 유지.
- **일관성 > 떡상:** 동일 톤·시그니처 인트로/아웃트로·규칙적 업로드가 장기 성장의 핵심.
- **메타데이터:** 제목(의문형)·설명 첫 2줄·태그·챕터·끝화면/카드로 회유.
- **자동 업로드:** YouTube Data API v3(resumable upload). 기본 일일 쿼터 10,000유닛, 업로드 1회 ~1,600유닛 → 하루 ~6편. 우리 페이스에 충분.
